import json, re, glob, os

base = os.environ.get("REG_REVIEW_ENGINE_V2_BASE_PATH", "C:/Projects/Regulatory-Review")
project_id = "<Project_A_id>"
out_dir = os.path.join(base, "data", "v2_dashboard_extracts", project_id)

# Collect all string leaves from every JSON file under out_dir
strings = []
def walk(node):
    if isinstance(node, dict):
        for v in node.values():
            walk(v)
    elif isinstance(node, list):
        for v in node:
            walk(v)
    elif isinstance(node, str):
        strings.append(node)

for path in glob.glob(os.path.join(out_dir, "**", "*.json"), recursive=True):
    with open(path, "r", encoding="utf-8") as f:
        walk(json.load(f))

# Normalize whitespace + lowercase both sides
concat = re.sub(r"\s+", " ", " ".join(strings)).strip().lower()
expected = "engine v2 lane 1 smoke text"
assert expected in concat, f"NOT FOUND. Sample: {concat[:200]!r}"
print("PASS -- VERBATIM JSON contains the expected phrase.")

## Step 13b -- AMBIGUOUS path / /files/exists polling (Findings 53, 58, 60, 65, 70, 75, 79)

This uses two coordinated mechanisms (per Finding 70 + 79):

- Server-side: `/files/complete` accepts `?testDelay=3000` ONLY when
  `process.env.VERCEL_ENV !== 'production'` AND
  `process.env.NODE_ENV !== 'production'`. Server commits INSERT, then sleeps.
- Client-side: under the same non-production test flag, UploadStep applies
  `AbortSignal.timeout(1000)` to the `/files/complete` fetch.

Procedure:
1. Trigger an upload with the test-delay flag.
2. After 1 second, fetch aborts -> AbortError -> UploadStep enters AMBIGUOUS.
3. UploadStep starts polling `GET /api/engine-v2/files/exists?project_id=<id>&file_id=<id>`
   at 1s intervals.
4. Within 5 polls, `/files/exists` returns 200 (the server INSERT committed at
   t=0, well within the 3s server delay).
5. UploadStep transitions to success WITHOUT calling `/files/orphan` and
   WITHOUT showing a cleanup-failure toast.
6. Verify the Storage object and DB row remain intact.

Test flag MUST be off in production (no AbortSignal.timeout, no testDelay
accepted).
