# Value Reconciliation Proposal - Matrix-Options SUBSTANCE_LIBRARY
**Status: PROPOSE-ONLY. No values have been changed. Every item below requires explicit HITL review/approval before any edit is applied to `src/lib/matrix-options/substanceLibrary.ts` or the catalog.**

## 1. Summary

Eight substance/endpoint pairs were researched against live authoritative sources (US EPA IRIS, Health Canada TRV v4.0, BC Protocol 28) and independently adversarially verified. All 8 verify passes returned **agree=true**, though one (vanadium pentoxide) adjusted the diagnosis label without disputing the underlying fact pattern. Three items are genuine wrong numbers currently wired into the library (copper, benzo[a]pyrene, lead) and require an explicit HITL value decision before correction. Two items are pure citation/label defects with no change to the numeric value (total PCBs/Aroclor 1254, chlorine dioxide recon artifact). Three items (phosphine, naphthalene, vanadium pentoxide) have catalog values already verified correct with nothing to fix on the number itself -- two are simply unwired gaps and one (naphthalene) needs no action at all. No autonomous value change is recommended anywhere in this batch; per this repo's "no default policy promotion" rule, the copper item additionally surfaces a 3-candidate default-selection decision (not just a number swap) for HITL.

## 3. SAFE TO CORRECT (label/citation only, no value change)

These items require no numeric change -- only a string/citation fix. Still HITL-gated per repo policy (no autonomous edits), but lower-risk since no downstream calculator output changes.

- **Total PCBs (Aroclor 1254) oral SF**: keep 2.0 per mg/kg-bw/day; rewrite the `sources` string in `substanceLibrary.ts` to disclose that the SF is borrowed from the generic US EPA IRIS PCBs (CASRN 1336-36-3) "high risk and persistence" tier, since Aroclor 1254 itself (CASRN 11097-69-1) has no IRIS carcinogenicity assessment. Current string ("US EPA IRIS Aroclor 1254...") implies an Aroclor-1254-specific IRIS cancer number that does not exist.
- **Chlorine dioxide**: the source catalog (`human_health_trv_values.json`) is already correctly labeled and requires no change. The defect is downstream, in `scripts/matrix-options/_recon/wire_candidates.json` and `wire_candidates_summary.md`, which collapse per-endpoint display_names onto a single substance-level field and happen to show the RfC's label next to the RfD's value. Recommend fixing the recon generator (and spot-checking other substances with both an RfD and RfC row, e.g. phosphine at recon row 44, for the same pattern) before it misleads a future wiring pass.

## 4. NEEDS HITL VALUE DECISION

These are the items where a NUMBER would change in `substanceLibrary.ts` if approved. No autonomous action taken; all three require explicit owner sign-off before edit.

- **Copper oral RfD**: proposed change from 0.04 to 0.426 mg/kg-bw/day (~10.6x increase). This is not a simple swap -- it is a default-policy decision among three sourced candidates: 0.426 (Health Canada TRV v4.0, approved, tier-1) vs 0.09 (BC Protocol 28 App 8A, needs_review) vs 0.141 (BC Protocol 28 App 8C, needs_review). HITL must decide which becomes the wired default; the 0.04 EPA RSL screening-level surrogate (SDWA Action Level back-calculation, not an IRIS toxicological RfD) is confirmed unsupportable as a default in any case.
- **Benzo[a]pyrene oral SF**: proposed change from 1.0 to 2.0 per mg/kg-bw/day (2x increase; this feeds cancer-risk calculations directly). The 1.0 value is a real EPA-published adult-only, non-ADAF alternate, not an error per se, but is not "the" IRIS default and its current citation does not disclose that distinction.
- **Lead oral RfD**: proposed change from 3.5e-3 to 5.0e-4 mg/kg-bw/day (7x decrease; more protective). Note for HITL: the Health Canada figure is a provisional "risk-specific dose" tied to a target blood-lead level (via the IEUBK biokinetic model), not a classic NOAEL/UF-derived RfD -- HITL should confirm this is compatible with how the library's `rfd_oral` field is used downstream before wiring it in as a like-for-like replacement. A BC Protocol 28 alternative (0.0006, needs_review) also exists.

## 5. Verifier Disagreements / Adjustments

No item had `agree: false`. One diagnosis-label adjustment is worth flagging:

- **Vanadium pentoxide**: the original researcher labeled this `library-value-wrong` (framing it as if a wrong number were already wired). The verifier agreed with every underlying fact (0.009 mg/kg-bw/day is a genuine, correctly-labeled IRIS oral RfD; the "Dermal" system-column label on the live IRIS page refers to the affected-organ system, not the exposure route) but corrected the diagnosis to `library-value-missing/unwired`, since `substanceLibrary.ts` has no `vanadium_pentoxide` entry at all (only an unrelated elemental "vanadium" key). This is a labeling correction only -- both agents agree the catalog value is right and the only real action is a new wiring, not a correction of an existing wrong number.

*All other items (copper, benzo[a]pyrene, lead, total PCBs, chlorine dioxide, phosphine, naphthalene) had full agreement on both diagnosis and correct value between the research and verify passes, with the verify pass independently re-fetching primary sources (IRIS PDFs, HC TRV PDFs, EFSA opinions) rather than trusting the original researcher's citations.*
