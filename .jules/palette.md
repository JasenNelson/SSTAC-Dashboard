## 2026-04-24 - Interactive Div Keyboard Accessibility\n**Learning:** In Next.js/React applications, using `div` elements with `onClick` handlers is a common pattern for custom interactive components (like expanding/collapsing sections), but it inherently lacks keyboard accessibility. This makes the UI completely invisible and unusable for screen reader and keyboard-only users.\n**Action:** When creating or modifying such components, always upgrade them to behave like native buttons by adding `role="button"`, `tabIndex={0}`, an appropriate ARIA state like `aria-expanded`, and an `onKeyDown` event listener that handles the 'Enter' and 'Space' keys (using `e.preventDefault()` to stop page scrolling for 'Space'). Additionally, apply visual focus indicators (e.g., `focus-visible:ring-2`) to ensure the focused state is clearly visible to sighted keyboard users.
