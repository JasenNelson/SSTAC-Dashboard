## 2026-04-26 - Invalid Nested Button Markup
**Learning:** Nesting a `<button>` tag inside another `<button>` tag is invalid HTML5, which triggers React hydration errors (`validateDOMNesting`), degrades the accessibility tree for screen reader users, and violates DOM parsing rules. This can happen easily when adding an inner interactive element (like an info or details toggle) inside a primary button.
**Action:** When a secondary interactive element must be nested inside a parent `<button>` (like the `<Users>` details toggle inside `LikeButton`), it cannot be a `<button>` tag. Instead, use a `<div>` or `<span>` and explicitly assign `role="button"`, `tabIndex={0}`, an `onKeyDown` handler for 'Enter' and 'Space' keys, and `aria-label` to provide an equivalent, accessible interactive control without breaking the HTML spec.
