## 2026-05-03 - [Sentinel] Replace weak Math.random() with secure cryptographic generation
**Vulnerability:** Found multiple instances of `Math.random()` being used to generate anonymous CEW user IDs (predictable tokens) and filenames for document uploads (predictable paths and potential collisions).
**Learning:** The existing implementation had an explicit requirement for backward compatibility of CEW anonymous session IDs (format: `authCode_timestamp_randomSuffix`). Replacing inline generation with a centralized utility that generated an entirely different format (32-character hex string) broke this backward compatibility and caused test failures. The proper approach is to maintain the exact existing string format structure but replace the `Math.random()` component with `crypto.randomBytes()` to secure the randomness without breaking data models or dependent logic.
**Prevention:** When refactoring legacy security or ID generation logic, always inspect adjacent comments for backward compatibility constraints and ensure the new secure implementation matches the expected output signature (length, format, character set) of the legacy implementation.
