## 2024-05-24 - [Replace Math.random() with cryptographically secure random generation]
**Vulnerability:** Use of `Math.random()` to generate security-sensitive values like session IDs, file names, and user IDs. `Math.random()` is not cryptographically secure and predictable, which could lead to session hijacking or ID collisions.
**Learning:** Found multiple instances where `Math.random()` was used for generating IDs and file names in API routes and components (`src/app/api/ranking-polls/submit/route.ts`, `src/app/api/review/upload/route.ts`, `src/app/api/wordcloud-polls/submit/route.ts`, `src/components/PollWithResults.tsx`, `src/components/Toast.tsx`). The memory instruction "When generating secure tokens, filenames, or user IDs... do not use the cryptographically weak `Math.random()`. Instead, use `crypto.randomBytes()` or the centralized `generateCEWUserId` utility" confirms this is a known risk pattern here.
**Prevention:** Always use Node's `crypto` module (`crypto.randomBytes()`) or `crypto.randomUUID()` when generating unique or secure identifiers, even for temporary session IDs or file names, to ensure unpredictability.
