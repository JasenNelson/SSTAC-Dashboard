## 2024-04-20 - Unnecessary dangerouslySetInnerHTML Usage
**Vulnerability:** Found `dangerouslySetInnerHTML` being used to render HTML entities (`&#8594;`, etc) inside a generic icon `div` inside `src/app/(dashboard)/bn-rrm/BNRRMClient.tsx`. This component received an `icon` prop as a string, introducing a Cross-Site Scripting (XSS) anti-pattern.
**Learning:** While in this specific instance it was fed hardcoded string literals, creating generic UI components that accept string props and inject them via `dangerouslySetInnerHTML` is an architectural risk. If future developers pass dynamic user content to the `icon` prop, it could result in an XSS vulnerability. Also, HTML entities passed as standard string props are properly handled by React when using literal Unicode characters.
**Prevention:** Avoid `dangerouslySetInnerHTML` unless rendering strictly sanitized HTML from a trusted source (e.g., using `isomorphic-dompurify`). For simple icons or symbols, use raw string literals of the Unicode characters or SVG icons.
