## 2025-04-16 - Prevent SQLite Parameter Interpolation Vulnerabilities
**Vulnerability:** SQL Injection via string interpolation in SQLite queries. The codebase used template literals to insert variables directly into SQL queries for LIMIT and OFFSET clauses (e.g., `LIMIT ${limit}`). It also used unsafe dynamic ordering with `ORDER BY ${orderBy}`.
**Learning:** Even if data appears structurally safe or is somewhat controlled, string interpolation into SQLite statements bypasses the underlying database's prepared statement features and opens up vectors for SQL injection. `better-sqlite3` supports parameterization for `LIMIT` and `OFFSET`. Dynamic `ORDER BY` clauses cannot use parameterization, so they must be strictly allowlisted at runtime.
**Prevention:** Always use parameterized queries (with `?`) for variables like `LIMIT` and `OFFSET`. For dynamic query components like `ORDER BY` and `UPDATE` column sets that cannot be parameterized natively by `better-sqlite3`, enforce a strict runtime allowlist validation against expected column names.
