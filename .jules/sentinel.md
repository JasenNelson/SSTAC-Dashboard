## 2025-02-14 - Predictable Randomness (Math.random) used for Secure IDs and File Names
**Vulnerability:** `Math.random().toString(36)` was used to generate suffixes for file uploads (`fileName` in `/api/review/upload/route.ts`) and anonymous user IDs (`finalUserId` in `/api/ranking-polls/submit/route.ts` and `/api/wordcloud-polls/submit/route.ts`), which is cryptographically weak and predictable, creating potential vulnerabilities like resource location guessing or ID collision/spoofing.
**Learning:** Even though Next.js API route handlers run on the server, developers sometimes fall back to familiar client-side/insecure methods like `Math.random` when trying to generate unique identifiers quickly, forgetting that it lacks cryptographic entropy.
**Prevention:** Always use Node's `crypto` module (`crypto.randomBytes(n).toString('hex')`) in API routes or server contexts, and `crypto.randomUUID()` in client contexts when generating secure tokens, file names, or user IDs.
