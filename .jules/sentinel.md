## 2025-05-22 - [Predictable Resource Location / IDOR Risk in File Uploads]
**Vulnerability:** The API endpoint for handling file uploads (`src/app/api/review/upload/route.ts`) was using `Math.random().toString(36)` to generate unique filenames. Since `Math.random()` generates cryptographically weak, predictable sequences, an attacker might guess the URLs/paths of uploaded review documents, leading to an Insecure Direct Object Reference (IDOR) or sensitive information disclosure.
**Learning:** In Next.js App Router applications handling file uploads to external storage (like Supabase), generating unique filenames requires cryptographically secure randomness to prevent enumeration attacks. The `crypto` module (Node.js) should be used on the server side instead of `Math.random()`.
**Prevention:** Always use `crypto.randomBytes(8).toString('hex')` or similar cryptographically secure functions for generating storage paths, file names, or user identifiers on the server side. Do not use predictable algorithms like `Math.random()` for any security-sensitive or unique identifier generation.
