## 2024-05-24 - [SQL Injection Risk in LIMIT/OFFSET and ORDER BY Clauses]
**Vulnerability:** Template literal interpolation was used for `LIMIT`, `OFFSET`, and `ORDER BY` clauses in `src/lib/sqlite/queries/index.ts`. While these may appear as simple numbers or limited strings in type definitions, runtime input might bypass TS type checking if data flows directly from external sources, leading to SQL injection.
**Learning:** `better-sqlite3` and modern SQLite correctly support parameterized queries (`?`) for `LIMIT` and `OFFSET` clauses. Additionally, since `ORDER BY` cannot be parameterized using standard placeholders, runtime whitelist validation must be strictly enforced before interpolating column names into the query.
**Prevention:** Always use parameterized placeholders (`?`) for all variables passed into queries, including `LIMIT` and `OFFSET`. For clauses that cannot be parameterized like `ORDER BY`, always enforce strict runtime whitelist validation against allowed columns to ensure safety.
