## 2024-06-25 - Removed dangerouslySetInnerHTML from FeatureCard
**Vulnerability:** XSS (Cross-Site Scripting) risk through `dangerouslySetInnerHTML` in the `FeatureCard` component within `src/app/(dashboard)/bn-rrm/BNRRMClient.tsx`.
**Learning:** React safely renders standard string props (including HTML entities like `&#8594;`) when passed as children (e.g., `<div>{icon}</div>`), automatically escaping them. Passing HTML entities into `dangerouslySetInnerHTML` is unnecessary and introduces potential XSS vectors if dynamic or user-generated data is later passed to the prop. To render characters correctly, pass the literal Unicode character (e.g., `→`) to the prop instead of the entity string, allowing React to render it natively as text without raw HTML injection.
**Prevention:** Avoid `dangerouslySetInnerHTML` unless explicitly rendering rich HTML content from a trusted, sanitized source. For symbols, icons, or standard strings, use literal Unicode characters directly in the React render cycle to benefit from built-in XSS protection.
